# 테스트 체크리스트

## ✅ 기본 기능 테스트

### 1. 메인 페이지 (`http://localhost:3000`)
- [ ] 페이지 로딩 확인
- [ ] 병원 목록 표시 확인 (6개 병원)
- [ ] 검색 기능 테스트
  - [ ] "서울" 검색 → 서울대학교병원, 삼성서울병원 등 표시
  - [ ] "Seoul" 검색 → 영어 이름으로 검색 확인
- [ ] 진료과 필터 테스트
  - [ ] "내과" 선택 → 내과가 있는 병원만 표시
- [ ] 도시 필터 테스트
  - [ ] "Seoul" 선택 → 서울 지역 병원만 표시
- [ ] 인기 진료과 버튼 클릭 테스트
- [ ] 병원 카드 클릭 → 상세 페이지 이동 확인

### 2. 병원 상세 페이지 (`/hospitals/[id]`)
- [ ] 병원 정보 표시 확인
- [ ] 진료과 목록 표시 확인
- [ ] 문의 폼 테스트
  - [ ] 일반 문의 전송
  - [ ] 예약 문의 전송 (희망 일시 포함)
  - [ ] 필수 필드 검증 (이름, 전화번호)
  - [ ] 다국어 선택 (한국어, 영어, 중국어, 일본어)
  - [ ] 성공 메시지 확인
  - [ ] 폼 초기화 확인

### 3. 관리자 대시보드 (`/admin`)
- [ ] 최근 병원 목록 표시 확인
- [ ] 최근 진료과 목록 표시 확인
- [ ] 각 링크 동작 확인
  - [ ] 병원 등록 링크
  - [ ] 진료과 등록 링크
  - [ ] 문의 내역 링크

## 🆕 새로 추가된 기능 테스트

### 4. 문의 내역 관리 (`/admin/requests`)
- [ ] 문의 목록 표시 확인
- [ ] 통계 카드 확인 (전체, 신규, 연락 완료, 완료)
- [ ] 상태 변경 테스트
  - [ ] 드롭다운에서 상태 선택
  - [ ] "신규" → "연락 완료" 변경
  - [ ] "연락 완료" → "완료" 변경
  - [ ] 상태 변경 후 즉시 반영 확인
  - [ ] 통계 카드 업데이트 확인
- [ ] 문의 상세 정보 확인
- [ ] 병원 링크 클릭 테스트

### 5. 병원 수정 (`/admin/hospitals/[id]/edit`)
- [ ] 병원 정보 로딩 확인
- [ ] 기존 데이터가 폼에 표시되는지 확인
- [ ] 병원명 수정 테스트
- [ ] 주소 수정 테스트
- [ ] 진료과 재선택 테스트
  - [ ] 기존 진료과 선택 해제
  - [ ] 새로운 진료과 선택
- [ ] 이미지 URL 수정 테스트
- [ ] 설명 수정 테스트
- [ ] 저장 후 목록에 반영 확인
- [ ] 취소 버튼 동작 확인

### 6. 병원 삭제
- [ ] 삭제 버튼 클릭
- [ ] 확인 다이얼로그 표시 확인
- [ ] 확인 후 삭제 확인
- [ ] 삭제 후 관리자 대시보드로 이동 확인
- [ ] 삭제된 병원이 목록에서 사라지는지 확인
- [ ] 삭제된 병원의 상세 페이지 접근 시 404 확인

### 7. 진료과 수정 (`/admin/departments/[id]/edit`)
- [ ] 진료과 정보 로딩 확인
- [ ] 기존 데이터가 폼에 표시되는지 확인
- [ ] 진료과명 수정 테스트
- [ ] 아이콘 변경 테스트
  - [ ] 기본 아이콘 선택
  - [ ] 직접 이모지 입력
- [ ] 설명 수정 테스트
- [ ] 저장 후 목록에 반영 확인

### 8. 진료과 삭제
- [ ] 삭제 버튼 클릭
- [ ] 확인 다이얼로그 표시 확인
- [ ] 사용 중인 진료과 삭제 시도
  - [ ] 에러 메시지 표시 확인
  - [ ] "이 진료과를 사용하는 병원이 있어 삭제할 수 없습니다" 메시지 확인
- [ ] 사용하지 않는 진료과 삭제
  - [ ] 삭제 성공 확인
  - [ ] 목록에서 사라지는지 확인
