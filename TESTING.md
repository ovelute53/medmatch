# 테스트 가이드

## 시드 데이터

시드 데이터가 성공적으로 생성되었습니다:
- ✅ 10개의 진료과 (내과, 외과, 정형외과, 산부인과, 소아과, 안과, 이비인후과, 치과, 피부과, 정신건강의학과)
- ✅ 6개의 병원 (서울대학교병원, 삼성서울병원, 아산서울병원, 세브란스병원, 강남세브란스병원, 부산대학교병원)

## 테스트 시나리오

### 1. 메인 페이지 테스트 (`http://localhost:3000`)

**검색 기능:**
- [ ] 병원명으로 검색 (예: "서울대")
- [ ] 영어 이름으로 검색 (예: "Seoul")
- [ ] 주소로 검색 (예: "강남")
- [ ] 검색 결과가 올바르게 표시되는지 확인

**필터링 기능:**
- [ ] 진료과별 필터링 (예: 내과 선택)
- [ ] 도시별 필터링 (예: Seoul 선택)
- [ ] 여러 필터 조합 테스트
- [ ] 필터 초기화 테스트

**인기 진료과 빠른 선택:**
- [ ] 각 진료과 버튼 클릭
- [ ] 선택된 진료과가 활성화되는지 확인
- [ ] 병원 목록이 필터링되는지 확인

**UI 테스트:**
- [ ] 로딩 스피너 표시 확인
- [ ] 에러 발생 시 에러 메시지 표시 확인
- [ ] 빈 상태일 때 EmptyState 표시 확인
- [ ] 반응형 디자인 확인 (모바일/태블릿/데스크톱)

### 2. 병원 상세 페이지 테스트 (`/hospitals/[id]`)

**병원 정보 확인:**
- [ ] 병원 기본 정보 표시 확인
- [ ] 진료과 목록 표시 확인
- [ ] 평점 및 리뷰 수 표시 확인
- [ ] 이미지 표시 확인 (있는 경우)

**문의 폼 테스트:**
- [ ] 일반 문의 전송
- [ ] 예약 문의 전송 (희망 일시 포함)
- [ ] 필수 필드 검증 (이름, 전화번호)
- [ ] 다국어 선택 (한국어, 영어, 중국어, 일본어)
- [ ] 성공 메시지 표시 확인
- [ ] 에러 메시지 표시 확인
- [ ] 로딩 상태 확인

### 3. 병원 목록 페이지 테스트 (`/hospitals`)

- [ ] 모든 병원이 표시되는지 확인
- [ ] 병원 카드 클릭 시 상세 페이지로 이동
- [ ] 평점 순 정렬 확인
- [ ] 빈 상태 처리 확인

### 4. 관리자 페이지 테스트 (`/admin`)

**대시보드:**
- [ ] 최근 병원 목록 표시 확인
- [ ] 최근 진료과 목록 표시 확인
- [ ] 각 링크가 올바르게 작동하는지 확인

**병원 등록 (`/admin/hospitals/new`):**
- [ ] 필수 필드 입력 (병원명, 주소)
- [ ] 진료과 선택 (다중 선택)
- [ ] 이미지 URL 입력
- [ ] 한국어/영어 설명 입력
- [ ] 저장 성공 확인
- [ ] 저장 후 목록에 표시되는지 확인

**진료과 등록 (`/admin/departments/new`):**
- [ ] 한국어/영어 이름 입력
- [ ] 아이콘 선택 또는 입력
- [ ] 설명 입력
- [ ] 저장 성공 확인

**문의 내역 (`/admin/requests`):**
- [ ] 문의 목록 표시 확인
- [ ] 통계 카드 표시 확인 (전체, 신규, 연락 완료, 완료)
- [ ] 문의 상세 정보 확인
- [ ] 병원 링크 클릭 테스트

### 5. API 테스트

**병원 조회 API:**
```bash
# 전체 병원 조회
curl http://localhost:3000/api/hospitals

# 검색
curl "http://localhost:3000/api/hospitals?search=서울"

# 진료과 필터
curl "http://localhost:3000/api/hospitals?departmentId=1"

# 도시 필터
curl "http://localhost:3000/api/hospitals?city=Seoul"
```

**진료과 조회 API:**
```bash
curl http://localhost:3000/api/departments
```

**문의 전송 API:**
```bash
curl -X POST http://localhost:3000/api/hospitals/1/requests \
  -H "Content-Type: application/json" \
  -d '{
    "type": "inquiry",
    "name": "홍길동",
    "phone": "010-1234-5678",
    "email": "test@example.com",
    "language": "ko",
    "message": "테스트 문의입니다"
  }'
```
